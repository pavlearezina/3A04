\section{Functional Requirements}
\label{sec:functional_requirements}
% Begin Section


You normally have two options for organizing your functional requirements:
\begin{enumerate}
	\item Organize first by \emph{business events}, then by \emph{viewpoints}
	\item Organize first by \emph{viewpoints}, then by \emph{business events}
\end{enumerate}
Choose the one which makes the most sense.

For example, if you wish to organization by business events:
\begin{enumerate}[{BE}1.]
	\item Business Event
	\begin{enumerate}[{VP1}.1]
		\item Viewpoint
			\begin{enumerate}
				\item Requirement
				\item Requirement
				\item \dots
			\end{enumerate}
		\item Viewpoint
			\begin{enumerate}
				\item Requirement
				\item Requirement
				\item \dots
			\end{enumerate}
		\item \dots
	\end{enumerate}
	\item Business Event
	\begin{enumerate}[{VP2}.1]
		\item Viewpoint
			\begin{enumerate}
				\item Requirement
				\item Requirement
				\item \dots
			\end{enumerate}
		\item Viewpoint
			\begin{enumerate}
				\item Requirement
				\item Requirement
				\item \dots
			\end{enumerate}
		\item \dots
	\end{enumerate}
\end{enumerate}

\underline{OR}, if you wish to organization by viewpoints:
\begin{enumerate}[{VP}1.]
	\item Viewpoint 
	\begin{enumerate}[{BE1}.1]
		\item Business Event
		\begin{enumerate}
			\item Requirement
			\item Requirement
			\item \dots
		\end{enumerate}
		\item Business Event

			\item Requirement
			\item Requirement

		\end{enumerate}
		\item \dots
	\end{enumerate}
	\item Viewpoint
	\begin{enumerate}[{BE2}.1]

			\item Requirement
			\item Requirement

		\end{enumerate}

		\begin{enumerate}
			\item Requirement
